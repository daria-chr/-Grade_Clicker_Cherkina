# Лабораторная работа №12. Жизненный цикл Activity (Activity Lifecycle)

# Цель работы:

изучить жизненный цикл Activity в Android и научиться правильно
обрабатывать переходы между состояниями. В результате лабораторной работы будет
создано интерактивное приложение "Grade Clicker" (Кликер оценок), которое
демонстрирует работу lifecycle callbacks и сохранение состояния при изменении
конфигурации устройства

## Ответы на контрольные вопросы

**Какие три метода вызываются при запуске приложения?**

При запуске приложения последовательно вызываются onCreate(), onStart() и onResume().

**Чем отличается onPause() от onStop()?**

onPause() вызывается, когда Activity теряет фокус, но может оставаться частично видимой (например, когда поверх неё появляется диалоговое окно). onStop() вызывается, когда Activity полностью скрывается с экрана — например, когда пользователь переходит в другое приложение или нажимает кнопку Home.

**Когда вызывается onRestart()?**

onRestart() вызывается только тогда, когда Activity возвращается на передний план после того, как была остановлена, но не уничтожена.

**Что происходит при повороте экрана?**

Поворот экрана является изменением конфигурации. Android полностью уничтожает текущую Activity, последовательно вызывая onPause(), onStop() и onDestroy(), а затем создаёт её заново через onCreate(), onStart() и onResume(). Если состояние не сохранено, все данные теряются.

**В чём разница между remember и rememberSaveable?**

remember сохраняет значение только во время рекомпозиций - пока Activity живёт. При пересоздании Activity значение теряется. rememberSaveable сохраняет значение и при рекомпозиции, и при изменении конфигурации - данные остаются даже после полного пересоздания Activity.

**Почему onCreate() должен вызывать super.onCreate()?**

Вызов super.onCreate() необходим, потому что родительский класс ComponentActivity выполняет в этом методе важную инициализацию: восстанавливает сохранённое состояние, настраивает систему жизненного цикла и подготавливает Activity к работе. Без этого вызова приложение завершится с ошибкой.

**Для чего используется класс Log?**

Класс Log используется для записи отладочных сообщений в консоль Logcat. Это позволяет отслеживать последовательность вызовов методов и поведение приложения в реальном времени. Основные уровни: Log.v() (verbose), Log.d() (debug), Log.i() (info), Log.w() (warning), Log.e() (error).

**Какой метод вызывается только один раз за время жизни Activity?**

Метод onCreate() вызывается ровно один раз - в момент создания Activity. В нём выполняется вся первичная инициализация: установка пользовательского интерфейса и начальные настройки.
